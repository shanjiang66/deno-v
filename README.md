# 🚀 Deno Deploy VLESS 节点极速部署指南 (自带网页伪装)

这是一个基于 Deno Deploy (Serverless 边缘计算平台) 构建的 VLESS + WS + TLS 代理节点项目。
相比于传统的 Docker 容器部署，本方案拥有 **0 秒冷启动、无闲置休眠、无需 UptimeRobot 保活** 的降维打击优势。

针对 Deno Deploy 日益严格的代码静态扫描和流量风控，本分支在原版高效率代码的基础上，**独家植入了深度网页伪装机制**。
* **原生拦截普通请求**：当查岗机器人或普通浏览器访问你的节点域名时，不再返回可疑的 `404` 或 `Bad Request`。

## 🛠️ 极速部署教程 (免分叉、纯网页操作)

使用 Deno Deploy 的 **Playground（游乐场）** 功能，无需建库，1 分钟即可点火上线。

### 第一步：获取代码并修改 UUID
1. 复制本仓库中的 `deno.tsx` 文件的全部代码。
2. 找一个文本编辑器打开，找到代码最上方第 6 行的配置项：
   \`\`\`typescript
   const userID = "填写你自己的真实UUID"; 
   \`\`\`
3. 将其替换为你自己的 UUID（例如 `93f6e6d0-...`）。

### 第二步：在 Deno Deploy 上线
1. 登录 [Deno Deploy 控制台](https://dash.deno.com/) (可直接用 GitHub 账号授权登录)。
2. 点击右上角的 **`New Project`**，然后选择正中间的 **`Playground`**。
3. 在打开的在线编辑器中，将左上角的文件名后缀改为 **`.tsx`**。
4. 清空编辑器里自带的代码，将你修改好 UUID 的代码**全部粘贴**进去。
5. 点击右上角蓝色的 **`Save & Deploy`** 按钮。
6. 部署成功后，你会获得一个专属的随机边缘域名（例如 `https://shiny-frog-12.deno.dev`）。**去掉 `https://`，这就是你的节点地址！**

## 💻 客户端配置参数 (以 v2rayN 为例)

Deno Deploy 原生提供可信的 TLS 证书加密，请在客户端中按照以下参数进行配置：

* **地址 (Address)**: `你的专属Deno域名` (如 shiny-frog-12.deno.dev)
* **端口 (Port)**: `443`
* **用户 ID (UUID)**: 填入你在代码中设置的 UUID
* **传输协议 (Network)**: `ws`
* **伪装类型 (Header type)**: `none`
* **路径 (Path)**: `/` (默认根目录)
* **底层传输安全 (TLS)**: `tls`
* **SNI**: `你的专属Deno域名` (与地址相同)

### 🔗 一键导入链接生成公式
将你的参数替换到下方模板中，直接复制整段字符串导入到客户端即可：
\`\`\`text
vless://填写你的UUID@填写你的Deno域名:443?encryption=none&security=tls&type=ws&host=填写你的Deno域名&path=%2f&sni=填写你的Deno域名#Deno_Serverless节点
\`\`\`
